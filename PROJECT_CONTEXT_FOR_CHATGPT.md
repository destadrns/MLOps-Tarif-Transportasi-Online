## 5. Source Code Summary

### `src/preprocessing.py`

**Purpose:** Data loading, cleaning, time feature creation, feature/target selection, and preprocessing pipeline creation.

**Important constants:**

- `TARGET_COLUMN = "price"`
- `REQUIRED_CAB_COLUMNS = ["price", "distance", "cab_type", "source", "destination", "name", "time_stamp"]`
- `MAIN_FEATURES = ["distance", "cab_type", "source", "destination", "name", "hour", "day", "month", "day_of_week"]`

**Important functions:**

- `load_raw_data(path)`: reads a CSV using pandas.
- `clean_cab_rides(df)`: validates required columns, removes missing `price`, removes `distance <= 0`, removes `price <= 0`, drops duplicates.
- `create_time_features(df)`: converts `time_stamp` from milliseconds to datetime with `errors="coerce"`, drops invalid timestamps, creates `hour`, `day`, `month`, `day_of_week`.
- `get_feature_target(df, include_surge=False)`: returns `X`, `y`, feature columns, numeric features, categorical features. Optional `include_surge` exists but main baseline uses `False`.
- `build_preprocessor(numeric_features, categorical_features, scale_numeric=True)`: builds `ColumnTransformer` with `StandardScaler` and `OneHotEncoder`. It includes compatibility fallback for `sparse_output=False` vs `sparse=False`.

**Key snippet:**

```python
def clean_cab_rides(df):
    missing_columns = [column for column in REQUIRED_CAB_COLUMNS if column not in df.columns]
    if missing_columns:
        raise ValueError(f"Missing required columns: {missing_columns}")

    cleaned = df.copy()
    cleaned = cleaned.dropna(subset=["price"])
    cleaned = cleaned[cleaned["distance"] > 0]
    cleaned = cleaned[cleaned["price"] > 0]
    cleaned = cleaned.drop_duplicates()
    return cleaned.reset_index(drop=True)
```

### `src/evaluate_model.py`

**Purpose:** Reusable evaluation and CT checklist utilities.

**Important functions:**

- `calculate_regression_metrics(y_true, y_pred)`: calculates MAE, RMSE, R2, mean actual price, median actual price.
- `create_prediction_result_dataframe(X_test, y_test, y_pred)`: creates result dataframe with actual price, predicted price, error, absolute error.
- `evaluate_by_group(result_df, group_column)`: calculates count, MAE, and RMSE by group.
- `create_ct_checklist(metrics, group_error_tables)`: creates CT checklist with thresholds.
- `save_report(path, content)`: writes report text.

**Important thresholds:**

- MAE <= 2.00.
- RMSE <= 3.50.
- R2 >= 0.85.
- Highest group MAE should not be extremely higher than overall MAE. The function uses `overall_mae * 1.5`.

**Key snippet:**

```python
def calculate_regression_metrics(y_true, y_pred):
    mae = mean_absolute_error(y_true, y_pred)
    rmse = np.sqrt(mean_squared_error(y_true, y_pred))
    r2 = r2_score(y_true, y_pred)
    return {
        "MAE": mae,
        "RMSE": rmse,
        "R2 Score": r2,
        "Mean Actual Price": float(pd.Series(y_true).mean()),
        "Median Actual Price": float(pd.Series(y_true).median()),
    }
```

```python
def run_all_validations(df):
    results = [
        validate_required_columns(df, REQUIRED_COLUMNS),
        validate_no_missing_target(df, target_column="price"),
        validate_positive_distance(df),
        validate_positive_price(df),
        validate_timestamp_convertible(df, timestamp_column="time_stamp"),
        validate_no_exact_duplicate_rows(df),
    ]
    results.extend(validate_no_missing_categorical_columns(df, CATEGORICAL_COLUMNS))
    return pd.DataFrame(results, columns=["Check Name", "Rule", "Failed Rows", "Status", "Recommendation"])
```

## 9. CD Scenario and API Summary

### CD Scenario

Source: `reports/cd_scenario_report.md`

CD means preparing the trained model so it can be safely released as a prediction service. This project does not deploy anything to real cloud services. The chosen strategy is Shadow Deployment.

**Shadow Deployment explanation:** The new model makes predictions in the background first. Users are not affected during the test period. Monitoring compares stability and error metrics before any promotion.

**Simulated release flow:**

1. Model v1.0 is trained and evaluated.
2. CI checklist validates data and preprocessing.
3. CT checklist validates performance and group-level errors.
4. Model is registered in `models/model_registry.json`.
5. Model is wrapped in a local FastAPI skeleton.
6. Shadow mode collects predictions in the background.
7. Monitoring checks MAE, RMSE, R2, and group-level MAE.
8. If stable, model can be promoted to active simulation.
9. If unstable, rollback to the previous approved model version.

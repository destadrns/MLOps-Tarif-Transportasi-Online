
# Member Role Distribution

Dokumen ini membagi peran presentasi untuk 5 anggota kelompok. Jika nama asli anggota sudah tersedia, ganti placeholder `Member 1` sampai `Member 5` dengan nama masing-masing.

| Member | Main Responsibility | Detail Penjelasan |
|---|---|---|
| Member 1 | Project overview, problem statement, dan dataset explanation | Menjelaskan latar belakang project, tujuan, dataset yang digunakan, target `price`, dan alasan project cocok untuk mini MLOps. |
| Member 2 | Data audit, cleaning, dan preprocessing | Menjelaskan hasil data audit, missing value pada `price`, proses cleaning, feature engineering waktu, dan alasan beberapa kolom dikeluarkan. |
| Member 3 | Model training dan evaluation | Menjelaskan model baseline, model terbaik Random Forest Regressor, metrik MAE, RMSE, R2, serta hasil error analysis. |
| Member 4 | CI dan CT quality checklist | Menjelaskan CI untuk validasi data/preprocessing dan CT untuk evaluasi performa serta stabilitas model. |
| Member 5 | CD scenario, API plan, shadow deployment, dan pipeline diagram | Menjelaskan model registry, FastAPI local simulation, Shadow Deployment, monitoring, rollback, dan diagram pipeline MLOps. |

## Catatan Presentasi

- Gunakan bahasa sederhana dan hindari klaim berlebihan.
- Tekankan bahwa project ini adalah simulasi pembelajaran.
- Jelaskan bahwa weather data belum digabungkan karena perlu alignment waktu dan lokasi.
- Setiap anggota cukup fokus pada bagian masing-masing agar presentasi tidak tumpang tindih.
